## Project Overview

This is **Part C** of a multi-part fraud detection project (학술대회 발표용). It handles smart contract deployment and token interaction on a local Ganache blockchain. The project uses an ERC-20-like token called **FraudDetectionToken (FDT)**.

## Architecture

The project is split into A/B/C parts (separate concerns). This repo is Part C (smart contract).

- `config.py` — Shared configuration (Ganache URL). Defined by Part A; imported by Parts B and C.
- `C_smart_contract/contracts/Token.sol` — Solidity ERC-20 token contract (FDT), compiled with solc 0.8.0.
- `C_smart_contract/deploy.py` — Compiles Token.sol via `solcx`, deploys to Ganache, saves ABI to `C_smart_contract/abi/Token.json` and deployment info to `C_smart_contract/deploy_info.json`.
- `C_smart_contract/interact.py` — Loads the deployed contract and performs token transfers between Ganache accounts.

## Prerequisites

- Python 3 with `web3`, `py-solc-x` packages
- **Ganache** running locally on `http://127.0.0.1:7545`

## Commands

```bash
# Deploy the contract (must run Ganache first)
python C_smart_contract/deploy.py

# Interact with deployed contract (run after deploy)
python C_smart_contract/interact.py
```

## Key Details

- Deploy must run before interact — `interact.py` reads `deploy_info.json` and `abi/Token.json` generated by `deploy.py`.
- Initial token supply is 1,000,000 FDT, minted to the first Ganache account (`accounts[0]`).
- All scripts use `sys.path.append` to import `config.py` from the project root.
